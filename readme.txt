Нечёткий поиск

Поиск слов в текте с возможными опечатками. Можно задать возможное количество
опечаток в слове ключом -s, возможные варианты:
    0 - без : слово в тексте находится лишь при полном совпадении
    1 - мало : мало опечаток, возможно по опечатке на каждые пять букв слова
    2 - много : много опечаток, возможно по опечатке на каждые три букв слова
    3 - неограниченно : совпадение любых слов

Запуск

В linux
    cat INPUT_WORDS | python3 search.py [-h] -t INPUT_TEXT [-o OUTPUT_FILE]
        [-a ALPHABET] [-as ALPHABET_WITH_SPACE] [-s SEARCH_ACCURACY]

В Windows
    type INPUT_WORDS | python search.py [-h] -t INPUT_TEXT [-o OUTPUT_FILE]
        [-a ALPHABET] [-as ALPHABET_WITH_SPACE] [-s SEARCH_ACCURACY]



Тесты написаны на модуль string_operations

Добавленна опция -f
Если заданна, то результат будет выводиться следующим образом:

<Искомое слово> <строка1>,<позиция1>[?] <строка2>,<позиция2>[?] ...
<строка1>,<позиция1> - строка и позиция первого найденного слова в тексте
[?] - выводится, если искомое и найденное слово совпали не полностью

Добавленна опция -с
Если заданна, то при поиске слов будет учитываться регистр

Добавленна опция -e, задающая кодировку файлов

Добавленна опция -p
Если заданна, то будет показываться прогресс поиска слов
